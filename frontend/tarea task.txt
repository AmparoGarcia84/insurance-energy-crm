## Campos de la tarea

### Relación con usuario asignado

* La tarea debe poder asignarse a un usuario existente
* Naming preferido:

  * `assignedToUserId` para la FK
  * relación Prisma: `assignedTo`

### Asunto

* texto corto
* nombre: `subject`

### Descripción

* texto libre largo
* nombre: `description`

### Selector entre proyecto o contacto

Este campo define el contexto principal de la tarea.

Nombre del campo:

* `contextType`

Valores:

* `PROJECT`
* `CONTACT`

### Prioridad

Nombre del campo:

* `priority`

Valores:

* `LOWEST`
* `LOW`
* `NORMAL`
* `HIGH`
* `HIGHEST`

### Recordatorio

Quiero una solución limpia para PostgreSQL + Prisma.

Campos propuestos:

* `hasReminder` Boolean
* `reminderAt` DateTime?
* `reminderChannel` enum?
* `reminderRecurrence` enum?

Canales:

* `EMAIL`
* `IN_APP`

Recurrencia:

* `NONE`
* `DAILY`
* `WEEKLY`
* `MONTHLY`
* `YEARLY`

Implementación inicial:

* el recordatorio puede ir dentro de `Task`
* salvo que veas una razón técnica fuerte para normalizarlo en tabla aparte desde ya

### Cliente opcional

* relación opcional con cliente
* nombres:

  * `clientId`
  * relación Prisma: `client`

### Selector del tipo de entidad relacionada

Quiero un campo para indicar con qué tipo de registro está relacionada la tarea.

Nombre:

* `relatedEntityType`

Valores:

* `CLIENT`
* `SALE`
* `QUOTE`
* `PROJECT`
* `SALES_ORDER`
* `PURCHASE_ORDER`
* `INVOICE`
* `CAMPAIGN`
* `SUPPLIER`
* `CASE`
* `CONTRACT`
* `PERSONAL_DATA`

Además:

* `relatedEntityId` como id del registro relacionado
* usa el mismo tipo de id que ya utilice el proyecto

La idea es una relación polimórfica a nivel aplicación:

* `relatedEntityType`
* `relatedEntityId`

### Fecha de vencimiento

* `dueDate`

### Estado

Nombre del campo:

* `status`

Valores:

* `NOT_STARTED`
* `DEFERRED`
* `IN_PROGRESS`
* `COMPLETED`
* `WAITING_FOR_INPUT`
* `UNLOGGED`

## Validaciones mínimas

* `subject` obligatorio
* `assignedToUserId` válido si se envía
* `clientId` válido si se envía
* `reminderAt` obligatorio si `hasReminder = true`
* `reminderChannel` obligatorio si `hasReminder = true`
* `reminderRecurrence` solo permitido si hay recordatorio
* `relatedEntityId` no puede existir sin `relatedEntityType`
* `relatedEntityType` no puede existir sin `relatedEntityId`

Quiero que distingas bien estos dos conceptos:

* `contextType`: define el marco principal de la tarea (`PROJECT` o `CONTACT`)
* `relatedEntityType`: define el tipo de registro concreto vinculado a la tarea

Aunque puedan parecer parecidos, no quiero que se mezclen conceptualmente.
